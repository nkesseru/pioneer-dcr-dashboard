## Who this is for

Anyone we hire (or rotate internally) into a Customer Success Manager role at Pioneer. Today that's one person; in two years it's a team of six.

It's also for *you* — the trainer. The templates and rubrics are how you keep coaching consistent as the team grows.

## How to use this repo

1. Read `00-academy-overview/README.md` end to end before anything else.
2. Work through pillars 01–07 in order during a new hire's first 90 days. Don't skip ahead — the order is intentional.
3. Use `templates/` for every lesson, drill, and certification. Don't reinvent the wheel each session.

## A note on tone

This is an operator's playbook. It's written like a senior teaches a junior: short sentences, real numbers, real conversations, real stakes. If you find a section that reads like a textbook, fix it.

## Maintenance

The Academy is a living document. Update it when:
- A real customer interaction taught us something the curriculum doesn't cover.
- A new PioneerOps feature changes how operators work.
- A graduate operator hits a gap we should close for the next class.
