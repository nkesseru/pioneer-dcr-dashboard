## Tools we use (current as of 2026-06)

- **Claude** — primary general-purpose assistant. Used for drafting, analysis, summarization.
- **Cursor / Claude Code** — code + script building. Used for small data work + internal tools.
- **Spreadsheet AI features** — for structured analysis.
- The current "right tool" list changes quarterly. Trainer maintains the active list.

Operator AI seat budget: ~$200/month per operator. Don't cheap out — the time saved pays back inside week one.

## The core insight

AI is a force multiplier, not a replacement. The trained operator:

- **Drafts faster** but always edits before sending
- **Analyzes faster** but always verifies the facts
- **Builds tools faster** but always knows what the tool is doing
- **Never** lets AI make a decision for them
- **Never** sends unedited AI output to a customer

An operator who turns into a "prompt button" is failing. The good operator gets to a better output, faster, with judgment intact.

## Mandatory artifact

Every operator builds a personal **Prompt Library** by end of week 6. Living document with their most-used prompts, by use case. Updated quarterly. Shared with the team.

## A working principle

If an AI tool saves you less than 30 minutes the first time you try it on a task, don't use it for that task. Stick to tasks where the leverage is real. Not everything benefits from AI; pretending it does wastes time.
