# Implementation Exercise Template

A practice deployment of PioneerOps using a real (or realistic) customer. The operator runs through the full implementation start-to-finish to demonstrate they can do it alone.

This is the bridge between knowing the platform (Pillar 03) and running an account (Pillar 05). Used for Operator I and Operator II certifications.

## The exercise — what the operator delivers

The operator must produce all of the following over 14 days. Each is a real implementation artifact, not a writeup about implementation.

### Day 1–2: Kickoff

- [ ] Kickoff call scheduled
- [ ] Kickoff deck built (customer-specific, not template)
- [ ] Kickoff call delivered (recorded if real customer)
- [ ] Kickoff notes captured in CRM
- [ ] 30-day plan shared with customer in writing

**Submit:** kickoff deck, recording (if available), 30-day plan doc.

### Day 3–5: Data + configuration

- [ ] Customer roster imported / configured
- [ ] Tech list imported / configured
- [ ] Service assignments created
- [ ] Role assignments set
- [ ] Mission Control configured for customer's CEO
- [ ] Customer aliases set up (if integrating QBO)

**Submit:** screenshots of configured customer, list of decisions made and why.

### Day 6–8: First-week training

- [ ] Field tech training session held (recorded)
- [ ] Admin training session held (recorded)
- [ ] CEO walkthrough scheduled

**Submit:** training session recordings, attendance log, follow-up doc sent to attendees.

### Day 9–12: First operational week

- [ ] First clock-ins captured in PioneerOps
- [ ] First DCR submitted
- [ ] First inspection completed
- [ ] First payroll cycle preview run
- [ ] Customer support tickets handled (if any)

**Submit:** screenshots of activation event metrics, troubleshooting log.

### Day 13–14: Wrap-up + handoff to steady state

- [ ] CEO walkthrough delivered
- [ ] First Mission Control review with CEO
- [ ] First QBR scheduled (90 days out)
- [ ] Activation event achieved (or honest report of why not)
- [ ] Internal handoff doc written (account in steady state)

**Submit:** CEO walkthrough recording, activation event report, handoff doc.

### Bar for Operator I certification

- All sections ≥ 2
- Activation event achieved within 30 days
- No customer escalation that required Senior intervention

### Bar for Operator II certification

- All sections ≥ 3
- Activation event achieved within 21 days
- At least one section at 4
- Customer NPS at end of implementation ≥ 50

## Trainer's questions during debrief

| Question | What you're listening for |
|---|---|
| What decision did you make this week that you weren't sure about? | Operator's judgment under uncertainty |
| What did the customer tell you that wasn't in the sales handoff? | Discovery skill |
| Where did PioneerOps fall short of what the customer needed? | Honest product feedback |
| What would you do differently in the next implementation? | Self-reflection |
| What did you learn that should go into the curriculum? | Continuous improvement contribution |

The last question is required. Every implementation exercise produces at least one curriculum update.

---

## Common failure modes

- ❌ Operator runs the kickoff like a sales demo
- ❌ Operator imports data without validating it against the customer's source
- ❌ Operator trains field techs but skips the admin
- ❌ Operator doesn't catch the activation event when it happens (or doesn't define it)
- ❌ Operator writes a handoff doc that no one else could use

---

## A note on "real vs simulated"

A real implementation is always better practice than a simulated one. If you have a real new customer at the right time, give it to the operator under supervision rather than running a fake exercise. The downside (slight risk of mistakes on a real account) is more than outweighed by the realism.

If you must simulate: pick a customer profile based on a recently onboarded account. Use real CRM history if available. Don't make the customer too easy or too hard — make them representative.
