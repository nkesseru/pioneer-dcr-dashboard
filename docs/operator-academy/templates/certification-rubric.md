Used for Apprentice / Operator I / Operator II / Senior / Lead certification reviews. Copy + customize per tier.

This rubric is the integrity layer. Reviewers commit to scoring strictly. The Academy's value depends on every "passed" certification meaning what it says.

> **Required by the [Constitution](../CONSTITUTION.md).** The Pioneer Test gate at the top of this rubric is a hard requirement. No tier promotion without an explicit Pioneer Test pass.

---

## THE PIONEER TEST — required gate

Before scoring any of the dimensions below, every reviewer must explicitly answer all four questions. If any answer is "No," the operator does not pass this tier — regardless of dimension scores.

- [ ] **Would this operator's customer work make Nick proud?**
- [ ] **Does this operator focus on business outcomes, not software features?**
- [ ] **Could this operator sit across from a cleaning company CEO and add real value, today?**
- [ ] **Would I trust this operator to represent Pioneer without supervision?**

If all four are "Yes" — proceed.

If any is "No" — stop. Document which gate failed and why. Continue feedback below for coaching purposes, but the tier does not change.

## Scoring scale (every dimension)

| Score | Meaning |
|---|---|
| 0 | Did not demonstrate |
| 1 | Demonstrated with significant gaps |
| 2 | Demonstrated competently |
| 3 | Demonstrated at level expected of senior operators |
| 4 | Demonstrated exceptionally — sets a new bar for the team |

Most dimensions should land at 2 or 3 for the relevant tier. Be honest. Inflated scores corrupt the next cohort's experience.

## Dimensions (assess each independently)

### 1. Implementation case study

- Real customer covered: Y / N
- Pre-Pioneer state documented: Y / N
- Three lessons + three improvements both included: Y / N
- Specific metrics, not vague claims: Y / N

**Score:** 0–4

**Comments:**

> What worked in this case study. What didn't.

---

### 2. Recorded customer kickoff

- Pre-call prep evident: Y / N
- Discovery questions, not survey: Y / N
- Customer-specific platform overview: Y / N
- Concrete implementation plan with dates: Y / N
- Next call scheduled before hangup: Y / N

**Score:** 0–4

**Comments:**

> Strongest moment of the call. Weakest moment.

---

### 3. Recorded QBR delivery

- Data accuracy (every number traceable): Y / N
- One narrative, not 12 unrelated stats: Y / N
- Three prioritized recommendations: Y / N
- CEO-appropriate language: Y / N
- Pushback handled cleanly: Y / N
- Time discipline: Y / N

**Score:** 0–4

**Comments:**

> What would a customer CEO take away from this QBR? Would they want a follow-up call?

---

### 4. AI-assisted analytical artifact

- Output quality (useful as-is): Y / N
- Prompt log shows iteration: Y / N
- Operator can identify what AI got wrong: Y / N
- 3× speedup vs unassisted documented: Y / N

**Score:** 0–4

**Comments:**

> What would you steal from this operator's workflow for your own?

---

### 5. Executive briefing (15 min)

- Strategic framing, not feature pitch: Y / N
- Real numbers (PioneerOps + industry): Y / N
- Three recommendations with ROI math: Y / N
- Hostile-but-fair Q&A handled cleanly: Y / N
- Time discipline (15 min, not 17): Y / N

**Score:** 0–4

**Comments:**

> Would a cleaning-industry CEO buy this operator's recommendations?

---

### 6. Difficult conversation

- Operator told the truth, including unflattering parts: Y / N
- Empathy for customer position evident: Y / N
- Operator held a position: Y / N
- Concrete next step left in place: Y / N

**Score:** 0–4

**Comments:**

> Did the operator move the relationship forward, or just survive the conversation?

---

### 7. Customer health model

- Real model in active use (not built for certification): Y / N
- Operator can defend every weighting: Y / N
- One real catch documented: Y / N
- One miss + lesson documented: Y / N

**Score:** 0–4

**Comments:**

> Would you give this model to a junior operator to start with?

## Overall decision

| Decision | When to apply |
|---|---|
| **Pass** | Pioneer Test passed (all 4 boxes) AND all 7 portfolio dimensions ≥ 2 AND all qualitative ≥ 3 AND tier-specific scorecard bars met |
| **Conditional pass** | Pioneer Test passed AND 6 of 7 portfolio dimensions ≥ 2 AND one specific gap to close in 30 days |
| **No pass — Pioneer Test fail** | Any Pioneer Test box "No." Tier blocked until specific gap closed. Often this is a coaching gap, not a portfolio gap. |
| **No pass — portfolio fail** | 2+ portfolio dimensions < 2, OR qualitative scores below tier bar, OR scorecard miss |

> Pass / Conditional pass / No pass

## What the operator gets

| Outcome | Materials |
|---|---|
| Pass | Written feedback (1 page), tier announcement, comp adjustment, account portfolio expansion |
| Conditional pass | Written feedback (1 page), specific remediation plan, retest date |
| No pass | Written feedback (1 page), 30/60/90 plan with explicit bars |

The written feedback is non-negotiable. Don't pass anyone with a verbal "good job." Don't fail anyone with a vague "needs work." Specifics every time.

## Reviewer reflection (post-decision)

For continuous improvement of the Academy:

- What surprised you about this operator's work?
- What gap in the curriculum did this certification expose?
- What rubric dimension is too easy / too hard for the tier?
- What would you change for the next certification?

Aggregate these across cohorts and update the rubric quarterly.
