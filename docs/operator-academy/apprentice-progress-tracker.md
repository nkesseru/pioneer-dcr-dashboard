# Apprentice Progress Tracker

> **Operational tracking, not curriculum.** A single Apprentice's path from Academy entry through Operator I certification and on to Operator II readiness, lived weekly.
>
> **Use this document as the canonical exemplar.** When a new Apprentice starts, copy [`templates/apprentice-progress-tracker-template.md`](./templates/apprentice-progress-tracker-template.md) into their own file. This file demonstrates the format with the first Apprentice's data (or a primed placeholder if no Apprentice yet).

> **Authority:** [Operator Certification Framework](./operator-certification-framework.md) Section 11 (Apprentice pre-certification status). Apprentice is a status, not a credential — automatic promotion to Operator I when the bar is met.
>
> **Custodian:** Academy lead. Updated weekly by the assigned trainer; reviewed jointly with the Apprentice at the Friday debrief.

## 17. Certification Gate Checklist

The pre-flight check before submitting [`certification-intent.md`](./templates/certification-intent.md). Every box must be checked AND defensible.

### Operator I Certification Gate

#### Knowledge gates
- [ ] All four foundation lessons read end-to-end
- [ ] Quiz scores ≥ 7/10 on at least 3 of the 4 lessons (record below)
- [ ] Constitution v1.1 read end-to-end
- [ ] Operator Glossary read in full pass
- [ ] Cert Framework read end-to-end

#### Practical gates
- [ ] 8 recorded Weekly Vital Reads (2 accounts × 4 weeks)
- [ ] At least 3 Loop entries created with verification dates
- [ ] At least 1 Loop entry fully closed (including any honest verified failure)
- [ ] At least 1 Customer Economics analysis produced
- [ ] At least 1 Pillar 8 verification pause documented
- [ ] At least 1 verified positive business outcome on an owned account

#### Role-play gates
- [ ] Passed at least 2 of 3 Scenario A role-plays (Customer Economics OR MC Diagnostics OR AI Operator)
- [ ] In each pass, the named trap was avoided

#### Evidence pack gates
- [ ] All universal evidence pack items present
- [ ] All Operator I tier items present
- [ ] At least 1 honest named failure documented
- [ ] Pioneer Test self-application completed

#### Doctrine gates
- [ ] Can recite the Pioneer Test (4 questions)
- [ ] Can name all six Operational Vitals
- [ ] Can describe the Loop (6 stages, in order)
- [ ] Can explain the Pillar 8 discipline in own words
- [ ] Can describe the Breadth/Judgment/Truth division of labor (Pillar 5)

### Operator II Certification Gate (stretch)

#### Cumulative bars
- [ ] All Operator I gates above (still cleared at higher bar)
- [ ] 8+ QBRs delivered (rubric pass on ≥ 6)
- [ ] 5+ Customer Economics analyses across distinct accounts
- [ ] 1+ Self-Sustaining Loop verified (customer reads solo)
- [ ] 1+ churn-risk save documented
- [ ] 1+ expansion deal closed
- [ ] 1+ Pillar 8 Discontinuity handled
- [ ] All Scenario A role-plays passed (all 4 foundation lessons)
- [ ] At least 2 Scenario B/C/D stretches passed

#### Business outcomes (12-month trailing)
- [ ] NRR ≥ 105%
- [ ] GRR ≥ 90%
- [ ] Customer NPS ≥ 50
- [ ] Health average ≥ 75

#### Portfolio artifact gates
- [ ] Implementation case study complete
- [ ] Executive briefing delivered (recorded)

- [ ] Customer health model in active use

## How to Use This Tracker

### When a new Apprentice starts

1. Copy [`templates/apprentice-progress-tracker-template.md`](./templates/apprentice-progress-tracker-template.md) into a new file: `docs/operator-academy/apprentice-progress-tracker-{lastname}.md`
2. Fill Sections 1–5 immediately (profile, start date, target, trainer)
3. Schedule the weekly cadence per Section 13
4. Begin updating Section 14 (Weekly Progress Log) every Friday
5. Update Section 15 (Competency Scorecard) monthly
6. Review Section 16 (Red/Yellow/Green Status) monthly with Apprentice
7. When all of Section 17 (Gates) are checked, submit Section 18 (Reviewer Signoff) starts the formal cert cycle

### When an Apprentice is certified

1. Lineage ID is assigned (Section 2 updated)
2. Registry entry is created in [`certification-registry.md`](./certification-registry.md)
3. This tracker is renamed to indicate certified Operator status (e.g., `apprentice-progress-tracker-{lastname}-OPI-graduated.md`)
4. Section 19 becomes the active section; new path-to-next-tier tracker may be started

### When an Apprentice does NOT certify

1. Specific gaps from Section 17 named in Academy lead's note
