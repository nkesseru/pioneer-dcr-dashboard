# 03 — PioneerOps Platform Mastery

The product itself. Operators must know every workflow, every screen, every common edge case — better than the customer ever will.

Maps to **Pillar 03** in the [Competency Model](../00-academy-overview/02-competency-model.md).

## Competencies covered

| ID | Competency | Target level by graduation |
|---|---|---|
| 03.1 | Core data model (techs, customers, assignments, sessions) | 3 |
| 03.2 | Time tracking + payroll workflows | 3 |
| 03.3 | Inspections + DCR systems | 2 |
| 03.4 | Communication threads + SMS transport | 2 |
| 03.5 | Mission Control + financial layers (Customer Economics, Financial Pulse) | 3 |
| 03.6 | Admin tools (roles, aliases, integrations) | 2 |

This is the largest pillar by competency count. Operators cannot fake their way through.

## Required hands-on work

Apprentice Operator:
- Tour every screen at least once during week 1
- Run a mock payroll cycle on a test customer
- Submit a test inspection + DCR end-to-end
- Configure a new test customer end-to-end

Operator I:
- Run a real payroll cycle for a small customer
- Submit a real inspection on an account they own
- Configure 2 new customer onboardings

Operator II:
- Diagnose a real platform issue without escalation
- Train a customer admin on Mission Control
- Run a Customer Economics walkthrough with a customer CEO
