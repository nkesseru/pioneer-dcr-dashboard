No doctrine changes. No certification framework changes. No lesson changes. The Operator I bar from framework §2 is the bar Nick must clear. The Academy must still require evidence.

1. Why a Founder Pilot

A Founder Pilot validates the Academy at the founder's own hand before subjecting an employee to it. Specifically:

  • Stress-tests the Academy. Nick walks every step and surfaces friction April would have hit. Friction becomes Framework v1.2 input.
  • Validates the bar's credibility. A founder who certifies to the same standard he wrote earns the credential's external validity. A bar held to oneself is more credible than a bar held only to employees.
  • De-risks April's cycle. Every gap Nick finds is closed before April starts. April's Cycle 2 is materially safer.
  • Establishes lineage. Registry entry POPS-OP-I-2026-001 becomes Nick Kesseru. April becomes POPS-OP-I-2026-002. Both real, both demonstrated.
  • Demonstrates honest economics. If Nick can't pass his own bar in 6 weeks, the bar may be too high — that's a finding worth knowing before April spends 30 hours.

If Nick discovers, during the pilot, that the bar is structurally unmeetable for a working operator, that becomes evidence for a framework adjustment AFTER Cycle 1 closes. Not during.

7. Validation outputs — what the Academy learns

The Founder Pilot is a validation exercise, not just a certification. At cycle end, the cycle record file documents:

  1. Bar realism. Was the Operator I bar achievable in 6 weeks by someone with deep prior experience? If yes, the bar's height is calibrated correctly for employees. If no, framework adjustment.
  2. Evidence pack friction. Where did even the founder struggle to produce the artifact in the format the evidence pack index asks for? Those friction points become Apprentice-friendly tooling.
  3. Reviewer scorecard usability. Did the reviewer scorecard work for Mike + Kirby + Reviewer 3? Where did they struggle?
  4. Recursion handling. Did the Pioneer Test reflection on one's own work pass the test? If not, the test needs a non-recursive variant for the founder case (framework v1.2 item).
  5. The 4-quadrant model. Was the classification useful? Should April's path use the same model? Does it require updating?
  6. Wall-clock floor verification. Did the 4-week Vital Read floor produce real evidence? Or did the founder cycle reveal that the floor is doctrinally pure but operationally arbitrary?

These outputs feed cycle-1-founder-pilot-record.md (authored Week 6 by Acting Academy Lead).

9. What this plan does NOT do

  • Does NOT modify any doctrine. Constitution + Framework + Financial Pulse + lessons unchanged.
  • Does NOT modify any certification standard. Same 9-category bar, same evidence pack, same Bootstrap rules.
  • Does NOT create a new certification tier. "Founder Pilot" is a candidate-context label, not a tier. Nick is pursuing Operator I.
  • Does NOT skip the Vital Read floor, Loop verification windows, or any other physical constraint.
  • Does NOT give Nick credit for past experience. Capability informs the path. Evidence is still required.
  • Does NOT replace the operator-i-launch-plan.md. The launch plan is the generic Operator I path. This document is the founder-context adaptation. Both stay; this one supersedes for Cycle 1 only.
