## Appendix B — Why there is no URL, no login, no portal

You asked about credentials and access path during the candidate-swap. Architecture rationale, moved to appendix:

The Academy is in `docs/operator-academy/` in this repo (`feature/academy-foundation-v1` branch as of 2026-06-11). Reading is your editor or GitHub. There is no web app, no login, no portal.

That is intentional. The Academy is meant to operate from the repo itself for the first ~3 cycles, with infrastructure added only after patterns repeat. Reasoning:

- A web app built before the manual flow is known is a web app built around the wrong workflow.
- Three cycles produce three real candidates, three real evidence packs, three real reviewer experiences — enough signal to design the right tooling.
- The Phase 2 move (after Cycle 3) is a small static site that serves these markdown files read-only. That's deferred until then.

For today, your access path is: this file. Everything you need is reachable from here in ≤ 2 hops.

A separate `founder-pilot-html.html` file in this same directory gives you a mobile-friendly UI for the Day 0 + Week 1 checklist if you prefer that to reading markdown. The HTML version reads the same content; it doesn't add new requirements.
