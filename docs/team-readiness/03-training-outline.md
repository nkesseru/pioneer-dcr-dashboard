# Deliverable 3 — Training Outlines

Two sessions per role:
- **Quick-Start (15 min)** — minimum viable to start working today
- **Full Training (60 min)** — every workflow they'll touch in their first month

Each outline assumes the trainee has already signed in to PioneerOps and is sitting in front of the live system. The trainer drives; trainee watches + repeats.

## New Cleaning Tech

### Quick-Start (15 min)

**Goal:** Tech can sign in, see today's work, clock in, finish a shift, submit a DCR.

| Minute | Topic | Action |
|---|---|---|
| 0–1 | Sign in | Open `/team-hub`, click **Continue with Google** (or use email/password if provisioned). Note: this email IS your Pioneer identity — keep it consistent. |
| 1–3 | Tour `/team-hub` | Top to bottom: Leadership messages → Announcements → My Hours → Requests & Support. "This is your home page." |
| 3–6 | Tour `/work` | Show today's assigned customer(s). Tap the customer card → see job details. |
| 6–10 | Clock in + clock out (LIVE PRACTICE) | Click **Clock In** on a real customer. Wait 60s. Click **Clock Out**. Observe the green active state + the timer. |
| 10–14 | Submit a practice DCR | Walk through Bathrooms / General / Kitchen / Office / Entry sections. Skip optional photos for the practice run. Sign and submit. |
| 14–15 | "If something goes wrong" | Show the **Need Help Now** card in Requests & Support. That's the lifeline. |

**Trainee leaves with:** Knowing how to sign in, clock in, clock out, submit a DCR.

---

### Full Training (60 min)

**Goal:** Tech can run an entire week independently — including My Hours self-check, time-adjustment requests, supply requests, call-outs, and reading communications.

| Block | Minutes | Topic |
|---|---|---|
| 1 | 0–5 | Sign in + identity hygiene (use the same email forever, never share password, what "signed in" means visually) |
| 2 | 5–15 | Team Hub deep tour. **Critical reading order each shift:** (1) From Leadership messages (2) Announcements (3) My Hours total |
| 3 | 15–25 | Work page + clock-in/out + DCR submission, full walkthrough with photos + customer signature path |
| 4 | 25–35 | My Hours — confidence chip, period total, pending corrections. Show how to expand "View All Shifts". Show how to submit a **time-adjustment request** end to end |
| 5 | 35–42 | Requests & Support — when to use Time Off vs Call Out vs Need Help Now. Submit each as a practice run if rules allow |
| 6 | 42–48 | Open Shifts — how to claim. Note: Rockstar bonus mints AFTER admin confirms coverage |
| 7 | 48–53 | Supply Station — gate codes, lock codes, **Start Supply Pickup** clock, submit a supply request |
| 8 | 53–58 | Communication messages — replying to a thread, acknowledging a recognition. **Reminder: replies are reviewed; this isn't Slack.** |
| 9 | 58–60 | What's OFF LIMITS — `/admin`, `/manager`, `/ceo`, `/inspections`. "If a link is shown to you and the page says 'Restricted', that's working as intended." |

**Skills checklist at end:**
- [ ] Sign in / sign out
- [ ] See today's assignments
- [ ] Clock in for cleaning
- [ ] Clock out + submit DCR with photos + signature
- [ ] Open and read a Leadership message
- [ ] Acknowledge or dismiss a Leadership message
- [ ] Verify My Hours total
- [ ] Submit a time-adjustment request
- [ ] Submit a call-out
- [ ] Submit a time-off request
- [ ] Claim an open shift
- [ ] Start + complete a Supply Pickup
- [ ] Submit a supply request
- [ ] Reply to a communication message

## New Inspector

> An inspector is an admin-tier user whose primary job is walks. In practice: April + Kirby today.

### Quick-Start (15 min)

**Goal:** Inspector can open `/inspections`, claim an overdue customer, run the form, submit, see the cycle close.

| Minute | Topic | Action |
|---|---|---|
| 0–2 | Sign in + access | Confirm `/inspections` loads (admin auth required). |
| 2–5 | Tour the page | Top to bottom: **Inspection Clock** → **Inspection Health** dashboard → **My Queue** → **Customer Registry**. Explain the five Total / Assigned / Completed / Overdue / Completion % tiles. |
| 5–7 | Start the inspection clock | Click **▶ Start Inspection**. Green hero turns on. Timer ticks. |
| 7–8 | Claim a customer | In Customer Registry → find an Overdue row → click **Assign to Me**. Row appears in My Queue. |
| 8–9 | Open the intake | In My Queue → **Open Inspection** on the same row → page jumps to Step 1 Setup with customer pre-filled. |
| 9–13 | Run the form | Step 1 confirm setup. Step 2 evaluate each item: Pass / Great / Fail / N/A. Fail = comment required. Skip optional photo. Step 3 Submit. |
| 13–14 | See the cycle close | Within 2 seconds, Customer Registry refreshes → that customer flips to Completed. |
| 14–15 | Clock out | Click **■ End Inspection**. Session writes paid time to payroll. |

**Trainee leaves with:** Knowing the full inspection flow plus how it bills time.

---

### Full Training (60 min)

| Block | Minutes | Topic |
|---|---|---|
| 1 | 0–5 | Inspection program purpose — 60-day cadence for every active customer, Pioneer's quality signal to CEO + customers |
| 2 | 5–10 | Page tour + auth confirmation; Inspection Clock vs. cleaning clock (singleton lock means one or the other, not both) |
| 3 | 10–20 | Run a full inspection on a real customer with full photo workflow + every verdict type used at least once (Pass, Great, Fail+comment, N/A) |
| 4 | 20–30 | Customer Registry — filter chips, status truth table (completed / assigned / overdue / unassigned), rotation suggestion explained ("try: Kirby" etc.), Take Over flow |
| 5 | 30–37 | Mark Complete (manual cadence-only closure) — when to use it (walk-by, phone confirmation) vs. Open Inspection (full score capture). Explain that Mark Complete does NOT create a score row. |
| 6 | 37–45 | Scoring math — Fail=0, Pass=1, Great=1.25, N/A excluded. All-Pass = 5.0 stars. ≥4.8 mints a Quality Win. Show a recent 5-star celebration. |
| 7 | 45–52 | Service Recoveries — when a customer issue or low inspection score warrants a recovery doc + assigned follow-up |
| 8 | 52–58 | Quality Wins + Pioneer Quality Score — how it feeds the morale dashboard on `/team-hub` and the CEO scorecard |
| 9 | 58–60 | Photo limits + "What we don't capture" — no candidate PII, no employee shaming. Inspections are objective; the system enforces it. |

**Skills checklist:**
- [ ] Start + complete an Inspection Shift (paid time logged)
- [ ] Assign to Me on an Overdue customer
- [ ] Take Over a customer assigned to someone else
- [ ] Release an assignment back to the queue
- [ ] Open Inspection + submit a full form
- [ ] Mark Complete a customer without filling the form
- [ ] Add a comment on a Fail (mandatory)
- [ ] Submit a 5-star inspection and confirm the Quality Win
- [ ] Create a Service Recovery for a sub-3 inspection

## New Office Manager

> Office Manager surface is `/manager`. Today: Kirby's primary cockpit. April uses it too.

### Quick-Start (15 min)

**Goal:** OM can open `/manager`, read the daily Action Required cockpit, triage one item, and submit the daily reflection.

| Minute | Topic | Action |
|---|---|---|
| 0–2 | Sign in + tour | Open `/manager`. Greeting card + Mission Control eyebrow. Verify role. |
| 2–4 | Read Action Required | The three counters (Critical / Attention / Healthy) + the per-bucket alert list = your day's priorities. |
| 4–7 | Triage one alert | Pick an Attention item — click into its source (e.g., a missing DCR → admin Labor Review). Resolve it. Refresh `/manager` to see it move. |
| 7–10 | Hiring Health card | The pill says Live GHL / Manual / No data yet. Show what each means. Optional: click **Refresh GHL** to manually re-pull. |
| 10–12 | Customer Health + Admin Health cards | Numbers come from sessions, callouts, supply, complaints. Explain each. |
| 12–14 | Submit Daily Reflection (LIVE) | Write 2–3 sentences. Submit. The doc gets written to `office_manager_reflections`. |
| 14–15 | Record Today's Bottleneck | Pick "Waiting on April / Customer / Vendor / Nobody". Optional note. Submit. |

**Trainee leaves with:** Knowing the daily 15-minute review ritual.

---

### Full Training (60 min)

| Block | Minutes | Topic |
|---|---|---|
| 1 | 0–5 | Where /manager fits — between tactical `/admin` and strategic `/ceo`. Daily cockpit for ops decisions. |
| 2 | 5–15 | Action Required cockpit in detail — what fires Critical vs Attention vs Healthy. How dismiss + snooze + suppress work in Mission Control alerts. |
| 3 | 15–22 | Communication Center — review open threads grouped by category. Reply to one. Close a resolved thread. Use **+ New Conversation** to start one. |
| 4 | 22–30 | Improvement Pipeline (`office_manager_improvements`) — submitted → approved → in_progress → implemented. Approve one. Move one. Show how completion bumps the CEO's "Improvements Advanced" tile. |
| 5 | 30–38 | Weekly Review form (Fridays only) — hiring numbers if no GHL snapshot, top wins, top concerns, action items |
| 6 | 38–45 | Hiring Health deep-dive — applicants 30d, interviews, working interviews, hires. When to update manually (the pill flips Manual). When the auto-sync runs (06:30 PT daily). |
| 7 | 45–52 | `/admin → Attendance` for call-out / time-off triage. `/admin → Schedule` for publishing the next 14 days. `/admin → Mission Control` alerts (shared with /manager) |
| 8 | 52–58 | Triage rhythm — morning 15 min on `/manager`, midday 5 min on `/admin → Payroll Exceptions`, end-of-day 5 min on `/admin → Tech Health` |
| 9 | 58–60 | Hand-offs — what to message April vs send to Nick vs handle alone. Leadership messaging modal walkthrough. |

**Skills checklist:**
- [ ] Read the Mission Control Action Required counters
- [ ] Triage an Attention alert end to end
- [ ] Submit Daily Reflection
- [ ] Log Today's Bottleneck
- [ ] Approve / reject an improvement
- [ ] Reply on a communication thread
- [ ] Start a new conversation with a tech
- [ ] Update Hiring weekly numbers (manual)
- [ ] Use Refresh GHL button
- [ ] Submit Weekly Review (on a Friday)
- [ ] Approve a Time Adjustment from Payroll Exceptions tab
- [ ] Run a payroll export (Verification Layer)

## New CEO / Executive

> Currently this is April + Nick. April runs `/ceo` daily; Nick checks in less often.

### Quick-Start (15 min)

**Goal:** CEO can read the company's health in ≤ 30 seconds and take one leadership action.

| Minute | Topic | Action |
|---|---|---|
| 0–2 | Sign in + tour | Open `/ceo`. Aesthetic note: luxury wellness, not enterprise dashboard. |
| 2–4 | Company Health score | Read the big % + 4 pillars. ≥ 92% = "steady", < 65 = "needs care". |
| 4–6 | Today's CEO Actions | 1–3 cards. Pick one. Click **Done** OR **Dismiss** OR **+ Task** depending on what fits. Watch the progress strip tick. |
| 6–9 | Quick Leadership Actions | Click **+ Recognize Employee**. Pick a teammate. Write 2 sentences of specific praise. Queue. Card confirms "Will appear next time they open Team Hub." |
| 9–11 | Inspection Program rollup | Read completion % + overdue count. If overdue > 3, this is the signal to nudge Kirby. |
| 11–13 | Open Conversations preview | Latest 3 communication threads. Click "Open in Manager →" to dive in. |
| 13–15 | Recent Activity feed | What you've done lately. Confirms the streak chip + Leadership Pulse. |

**Trainee leaves with:** A daily ritual: open `/ceo`, read 30 sec, take ONE leadership action.

---

### Full Training (60 min)

| Block | Minutes | Topic |
|---|---|---|
| 1 | 0–6 | Why `/ceo` exists — leadership signal, not operational triage. /manager is for triage. |
| 2 | 6–16 | Company Health score formula — how the 5 pillars (Quality, Staffing, Service, Customer, Hiring) get computed. Calibrate "what looks right" for current Pioneer state. |
| 3 | 16–25 | Today's CEO Actions — the 6 auto-suggestions (coverage gap, complaint > 48h, repeat customer issue, hiring stalled, approvals backlog, no recognition this month). Practice clicking Done vs Dismiss vs + Task for each kind. |
| 4 | 25–32 | Leadership messaging — three modes (Recognize / Team / Office Manager). Working-hours protection: techs see immediately on next sign-in; OM messages outside 8 AM–6 PM PT queue until 8 AM. |
| 5 | 32–40 | Department Scorecards — what each metric means, what number is "good" right now |
| 6 | 40–48 | Attention Needed vs Momentum — guiding tone, NOT alarm. April's role is to lead, not to put out fires |
| 7 | 48–53 | Recent Activity + Leadership Streak — the streak resets if no action AND no message in a day. Use it as a personal accountability nudge. |
| 8 | 53–58 | Open Conversations preview → /manager Communication Center handoff. When to read in /ceo vs go to /manager. |
| 9 | 58–60 | What CEO Mission Control deliberately does NOT include — financial widgets, payroll detail, raw operational alerts. Those live elsewhere. |

**Skills checklist:**
- [ ] Read the Company Health % + interpret each pillar
- [ ] Use Today's CEO Actions (Done / Dismiss / + Task) on every kind of suggestion
- [ ] Send a recognition to a specific employee
- [ ] Send a team broadcast
- [ ] Send an office manager note
- [ ] Confirm working-hours queueing behavior (try a 9 PM PT compose to OM)
- [ ] Open Inspection Program rollup and confirm it matches /inspections
- [ ] Open Conversations preview → land in /manager Communication Center
- [ ] Maintain a Leadership Streak across 3 consecutive days
